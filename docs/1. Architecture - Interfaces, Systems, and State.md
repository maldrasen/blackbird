The Controller portions of the system are further split into Interfaces, Systems, and States. This split is to make testing easier, as the Systems are designed to be independent from the views. Each system has an associated State object that handles all the internal state for the systems as both the interfaces and systems are themselves stateless.

### Interface
An interface acts as a bridge between the views and the systems.

### Systems
Stateless, isolated, testable, business logic. 

### State
Global objects of state stuff.
