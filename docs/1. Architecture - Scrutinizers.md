Like the Weaver and Looms we need a way to easily define requirement objects. The CentralScrutinizer works like the Weaver, taking in the current context, and an array of requirements, checking to make sure that all the requirements pass. (We could also add an or mode as well, to check if any requirement passes, though we haven't needed that yet.) Like the Weaver, the Scrutinizers have a simple internal DSL.

#### Cock Scrutinizer
- `A:has-cock` Character A has a cock component.
#### Breast Scrutinizer
- `A:has-breasts` Character A has a breasts component.
- `A:breasts-at-least-<size>` Minimum breast size requirement.
#### Pussy Scrutinizer
- `A:has-pussy` Character A has a pussy component.
#### Equipment Scrutinizer
- `A:equipment.not-naked` Character has some clothing equipped.
#### Other
- `A:unbound` Character is not bound in any way. (I'll probably change this once I've actually designed some kind of bondage system as we'll need to differentiate the ways in which a character can be bound. This is mostly a placeholder for now.)
